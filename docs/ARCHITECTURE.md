## 一、项目概述

### 1.1 项目目标
设计一个能够根据图片和文字要求生成短视频剧本的 Agent 应用，支持：
- 图片上传（单张或多张）
- 文字输入
- 调用火山引擎模型 API 生成剧本
- 多轮对话迭代优化

### 1.2 用户群体
- 非技术人员
- 需要快速生成短视频剧本的用户

### 1.3 技术选型原则
- **保持 Python 生态**：优先使用 Python 技术栈，保持与主流 AI 框架的兼容性
- **前后端分离**：解耦前端界面和后端逻辑，提高可维护性
- **美观与实用平衡**：在保证功能的前提下，提供良好的用户体验
- **易于分发**：支持多平台打包，方便非技术人员使用

## 十、常见问题

### 10.1 为什么选择前后端分离？

1. **UI 美观性**：可以完全自定义界面，不受 Gradio 限制
2. **技术灵活性**：前端可以用任意框架，后端保持 Python
3. **易于扩展**：一套后端支持网页/App/桌面多端
4. **团队协作**：前端和后端可以并行开发

### 10.2 为什么选择 Electron？

1. **成熟稳定**：业界广泛使用，打包工具完善
2. **跨平台**：一套代码打包 Windows/macOS/Linux
3. **生态丰富**：有大量现成的工具和模板
4. **易于调试**：前端代码可以在浏览器中直接调试

### 10.3 为什么保持 Python 后端？

1. **AI 生态**：主流 AI 框架（LangChain、AutoGPT）都是 Python
2. **开发效率**：Python 语法简洁，开发速度快
3. **社区支持**：大量开源项目和教程

### 10.4 打包大小问题

| 方案 | 打包大小 | 说明 |
|------|----------|------|
| Gradio + PyInstaller | ~150MB | 最小 |
| Electron + Python | ~300MB | 适中 |
| Electron + Node.js | ~200MB | 可以考虑 |

**建议**：接受 Electron 方案稍大的体积，换取更好的用户体验
